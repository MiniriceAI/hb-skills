## 核心定位

商单文案的核心矛盾：用户来看内容，品牌来买曝光。好的商单让用户觉得"这是在帮我"，让品牌觉得"这在推广我"，两者不冲突。核心技术：**内容价值优先，植入自然，7:3节奏把控**。

---

## 触发场景

- "帮我写一篇[品牌]的商单"
- "我接了[产品]的广告，需要一篇小红书/微信推文"
- "帮我写一篇合作推广文章"
- 任何明确包含品牌/产品推广目的的内容创作需求

## 工作流

### 步骤1：Brief解读与信息收集

开始写作前，必须明确以下信息（若用户未提供，逐一追问）：

**核心信息（必须知道才能开始）**：
- 品牌/产品名称
- 产品核心卖点（品牌方给的USP，通常1-3条）
- 目标受众（品牌方指定的人群描述）
- 发布平台（小红书/微信公众号/抖音脚本/视频号）
- 字数要求或时长要求
- 软广还是硬广（是否要求明确露出品牌名/产品名）

**补充信息（有则更好）**：
- 必须提及的硬性要素（活动信息/价格/购买渠道说明）
- 禁止出现的内容（竞品名/特定词汇/夸大宣传词）
- 博主定位（美妆/职场/生活方式/母婴，直接决定写作视角）
- 是否有真实使用体验可参考？

### 步骤2：角度选择

根据产品特性和受众选择主角度（辅以一个次角度）：

**角度A：问题-解决方案型**
适用：解决明确痛点的产品（护肤/效率工具/健康品）
逻辑：我有X问题 → 试了很多方法没用 → 发现这个产品 → 具体效果
风险：容易显假，场景必须足够真实和具体

**角度B：生活方式融入型**
适用：品牌调性产品（轻奢/美学/食品/香氛）
逻辑：我的某种生活状态 → 这个产品是这种生活的一部分 → 为什么它适合同类人
风险：植入感淡，但品牌记忆度高，适合长线合作

**角度C：对比验证型**
适用：强功能性产品（数码/家电/功效护肤）
逻辑：我之前用[同类产品]，现在换了这个，区别是[具体对比数据/感受]
风险：需要真实对比细节，否则显空洞

**角度D：专业/权威型**
适用：成分/技术驱动的产品（医美/功能食品/专业工具）
逻辑：从专业视角解释产品背后的原理，然后评价产品实现了这个原理
风险：需要真实的专业知识支撑，不能只喊概念

**角度E：场景事件型**
适用：礼品/节日产品/限定款/季节性产品
逻辑：因为某个具体事件/场景 → 这个产品是完美选择 → 具体推荐
风险：场景必须高度具体，否则读者无法代入

### 步骤3：结构组装（7:3原则）

全文70%是有价值的独立内容，30%是产品相关信息。

读者视角：文章感觉是"有用内容里自然提到了一个产品"，而非"广告里加了一些内容"。

**植入节奏**：
```
[纯内容部分，建立价值和信任] → [自然过渡] → [产品出现] → [核心产品信息，1-2个卖点] → [回归内容或行动引导]
```

**自然过渡的4种写法**：
- 场景过渡："正好最近在用X，顺便说说真实感受..."
- 解决问题过渡："后来找到了一个方案，就是..."
- 长期使用过渡："用了半年了，这是我的真实体验..."
- 朋友推荐过渡："被好几个人安利，然后自己试了..."

### 步骤4：平台格式执行

严格按对应平台规范输出（见下方模板）

## 输出格式

### 小红书商单模板

**标题格式**：20字以内，含核心关键词+情绪词
```
[痛点/场景]+[产品类别]+[结果/感受]
示例："干皮救星！这款精华用了3周皮肤喝饱水的感觉"
```

**正文结构**：
```
[开头，3-5行]
场景代入或痛点共鸣，不提产品名，结尾留悬念

[过渡，1-2行]
自然引出产品

[产品体验，3-5行]
具体使用感受，1-2个核心卖点，给出具体细节（时间/部位/变化）

[使用建议，2-3行]
tips或适合人群

[互动引导，1行]
引导评论的问题

[标签，5-8个]
#产品类别 #功效词 #场景词 #人群词
```

格式规范：每段不超过3行，用空行分隔，表情符号点缀非堆砌，总字数300-600字

---

### 微信公众号商单模板

**标题**：20-40字，悬念/利益点/反常识

**文章结构**（总字数1000-2000字）：
```
引入（150-300字）：话题/故事/问题，与产品使用场景相关，不提品牌

内容主体（400-800字）：有独立阅读价值的内容，可以不看广告也值得读

产品植入（200-400字）：
  - 自然引出，说清楚为什么选这个而不是其他的
  - 重点介绍1-2个核心卖点（不是全部卖点）
  - 给出具体使用方法或购买信息

结尾互动（50-100字）：引导评论或分享
```

格式规范：小标题分割内容，每节200-300字；文章开头或结尾标注"广告"或"品牌合作"（合规要求）

---

### 抖音/视频号口播脚本模板

**视频时长**：60-90秒

**脚本结构**：
```
[0-5秒] 钩子：
一句话抓住注意力（提出问题或反常识陈述）
[镜头提示：正面直视，表情专注]

[5-20秒] 共鸣建立：
展开问题，让观众点头认同
[停顿节奏：每句话后自然停0.5秒]

[20-45秒] 内容价值：
有用信息，自然带出产品名
[产品出现时机标注：第X秒]

[45-65秒] 产品介绍：
只说一个核心点（不超过2个卖点）
具体说使用体验，不说空洞形容词
[产品特写画面时机：第X-X秒]

[65-80秒] 行动引导：
告诉观众下一步怎么做
[结尾镜头提示]
```

格式规范：口语化，短句每句不超过15字，避免书面语，每句话后标注[停顿/语气]提示

## Gotchas

**最常见的商单失败原因**

1. 植入太硬：前三句在聊生活，第四句突然"今天要给大家推荐一款产品" → 用户失去信任，直接关掉
2. 卖点堆砌：把品牌方给的所有卖点全写进去 → 读者记不住任何一个，只选最重要的1-2个
3. 假体验感：说"我用了很久"但描述太泛 → 加入具体时间、具体场景、具体感受细节
4. 与博主定位不符：财经博主推护肤品，美妆博主推金融产品 → 即使强行找到角度也会显得奇怪

**合规提醒（不可忽略）**
- 国内平台要求标注广告合作，不标注是违规行为
- 不允许使用绝对化用语（最好/第一/顶级/根治）
- 医疗健康类产品有广告法特殊限制，不能承诺疗效

**品牌方给的文案不能直接用**
品牌方提供的USP是出发点，不是最终文案。必须转化成用户语言，而非品牌语言。
"科技感"→ 用户语言是"用这个功能，我实际节省了X时间"

**7:3比例的灵活应用**
- 软广要求高：可以到8:2甚至9:1，产品几乎不出现在主体内容中
- 硬广要求时：至少保持6:4，不要低于5:5
- 转化目标（要求引流购买）：可以到6:4，产品信息部分可以更详细具体
