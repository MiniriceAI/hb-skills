# 知识宫殿 - 战略认知更新流程

## 与其他记忆层的边界

知识宫殿不是默认记忆库，而是**精选后的长期认知库**。

- **M-flow**：放原始会话重点、历史上下文、近期问题与解法
- **OpenViking 共享记忆**：放多个 agent 之间需要共用的稳定事实
- **知识宫殿**：只放高价值、经筛选、适合长期复用的认知

### 不该直接放进知识宫殿的内容

- 完整 session 流水账
- 普通问答记录
- 一次性操作步骤
- 还没验证过的临时想法

### 适合提升到知识宫殿的内容

- 已验证的重要洞察
- 稳定可复用的工作流
- 战略级决策与原因
- 会反复拿来指导后续行动的经验和教训

## 执行流程

### Step 1: 扫描待分析文件

1. 读取 `YOUR_LOCAL_PATH/Desktop/MD文件/` 目录下所有 `.md` 文件
2. 显示文件列表（文件名、修改时间）
3. 如果没有新文件，告知用户并结束流程

**输出格式**:
```
📂 发现 N 个待分析文件：
1. [文件名] - [修改时间]
2. ...
```

### Step 2: 深度分析文件

对每个文件进行分析，参照 `核心认知.md` 的现有内容：

**分析维度**:
1. **合并同类项**: 识别与现有板块重叠的内容，标记需要合并的位置
2. **新增洞察**: 发现全新的战略思考，初步评估其价值
3. **打分评估**: 使用打分系统评估每条洞察
4. **冲突检测**: 识别与现有认知冲突或矛盾的观点

**打分系统标准**:
| 分数范围 | 星级 | 含义 |
|---------|------|------|
| 100 | ⭐⭐⭐ | 核心战略、立即重要、经验验证、可直接执行 |
| 95-98 | ⭐⭐⭐ | 重要战略、已验证、高执行价值 |
| 90-92 | ⭐⭐ | 重要指导、有参考价值、需要调整 |
| 85-88 | ⭐ | 参考级、局部适用、需要结合场景 |
| <85 | 无 | 参考信息，不纳入核心认知 |

**分析方法论**:
- **双层结构识别**: 区分战略层（面）和执行层（点）
- **即时反馈识别**: 捕捉核心反应中的重要观点

### Step 3: 高分内容呈现 + 交互式决策

**呈现格式**（先给概览）:
```
📋 本次分析发现 N 个需要决策的问题：

1. [主题1] - [核心主旨1句话]
2. [主题2] - [核心主旨1句话]
...

现在开始逐一讨论：
```

**需要决策的内容类型**:
- **观点冲突**: 新内容与现有认知冲突，需要确认保留哪个
- **全新思考**: 全新的战略内容，需要确认是否纳入、归入哪个板块
- **细节补充**: 重要工作细节不完整，需要用户补充或明确

**交互规则**:
- 每个问题深入讨论，直到答案清晰
- 如果用户回答涉及重要延伸信息，主动追问
- 确认一个问题后，再转下一个
- 使用 AskUserQuestion 工具进行结构化提问

### Step 4: 生成 Memo + 更新索引

**Memo 格式**:
```
---
## 📝 本次更新 Memo（YYYY-MM-DD）

### 核心重点
- [重点1]
- [重点2]

### 新增/修改的认知条目
| 条目 | 分数 | 操作 | 归属板块 |
|------|------|------|---------|
| xxx | 95分⭐⭐⭐ | 新增 | xxx |

### 处理的文件
- [文件名] → 归档至 [目录]
---
```

**索引更新**:
编辑 `文件索引.md`，添加本次处理记录

### Step 5: 更新核心认知 + 文件归档

**核心认知更新规则**:
1. 按分数高低插入新洞察（100分在前）
2. 同分数按日期排序（最新在前）
3. 合并同类项时保留更完整的版本
4. 更新优先级清单（P0/P1/P2）如有必要

**洞察条目格式**:
```markdown
### 主题名称（参与者, YYYY-MM-DD, 分数⭐级别）

#### 核心洞察
- 要点1
- 要点2

#### 决策/执行要点
- 具体行动项

---
```

**文件归档映射**:
| 内容主题 | 归档目录 |
|---------|---------|
| 战略框架、认知模型 | `01-核心认知/` |
| 商业模式、运营实践 | `02-业务实战/` |
| 内容创作、爆款方法 | `03-内容创作/` |
| AI工具、技术方案 | `04-技术工具/` |
| 风险管控、合规策略 | `05-风控合规/` |
| 对外合作、渠道策略 | `06-合作策略/` |
| 个人复盘、认知升级 | `07-个人思考/` |
| 数据分析、财务模型 | `08-运营数据/` |
| 会议纪要 | `09-会议记录/` |
| 成功案例、模板 | `10-爆款案例/` |

**⚠️ 归档操作（极端重要）**:
- **正确路径**: `YOUR_KNOWLEDGE_BASE_PATH/[对应子文件夹]/`
- **错误路径**: ❌ 不要放在 `YOUR_LOCAL_PATH/Desktop/MD文件/已处理/` 或其他任何临时目录
- 使用 Bash 的 `mv` 命令直接移动到知识宫殿对应的子文件夹
- 示例: `mv "YOUR_LOCAL_PATH/Desktop/MD文件/xxx.md" "YOUR_KNOWLEDGE_BASE_PATH/09-会议记录/"`

## 核心原则

1. **极简执行**: 用户只需把文件放到待分析目录，说一句"继续整理"
2. **交互式决策**: 重要判断需要用户确认，不擅自决定
3. **价值优先**: 只纳打分90分以上的战略级认知
4. **归档到位**: 处理完的文件必须移到正确的知识宫殿子目录

## Gotchas（踩过的坑）

1. **待补充** - 这个技能的具体坑点需要根据实际使用经验补充
2. **待补充** - 继续添加实际遇到的问题
3. **待补充** - 至少需要3个具体的Gotchas条目

|---------|------|------|-----------------|
| 100 | ⭐⭐⭐ | 核心战略、立即重要、经验验证、可直接执行 | ✅ 是 |
| 95-98 | ⭐⭐⭐ | 重要战略、已验证、高执行价值 | ✅ 是 |

3. **执行价值** - 是否有明确可执行的动作
4. **时间敏感性** - 是否需要立即行动
5. **可复用性** - 能否复用于类似场景

## 常见场景判断

### 需要决策的问题类型

1. **观点冲突**: 新内容与现有认知矛盾
2. **全新思考**: 首次出现的战略方向
3. **细节补充**: 需要用户确认具体数字/时间
4. **打分争议**: 90分边界附近的判断

### 不纳入核心认知的情况

- 用户明确说"不重要"、"不用记"
- 分数低于90分
- 纯执行层面的临时信息
- 过时且已更新的内容
