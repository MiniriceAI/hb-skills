# 核心定位

对单只美股进行系统性的多维度分析，覆盖基本面、技术面、催化剂、竞争格局和估值，最终输出结构化研究报告（含明确的多空理由）。不是预测涨跌，而是帮助用户理解"这只股票值多少钱、风险是什么、当前市场怎么看"。

---

# 触发场景

- 用户输入股票代码（如"帮我分析 NVDA"）
- 用户询问某只股票是否值得买入/持有/卖出
- 用户提供了季报数据，需要解读
- 用户需要对比两只股票（先分别分析，再对比）

# 工作流

## Step 1: Intake

收集以下信息（若用户未提供，使用默认值）：

```
代码：[用户提供，若只给公司名则先确认代码]
分析视角：长线（>1年）/ 波段（1-3个月）/ 短线（<1个月）→ 默认长线
持仓状态：已持有 / 考虑买入 / 空头研究 → 影响报告侧重点
重点关注：基本面 / 技术面 / 特定事件（如即将财报）
```

## Step 2: 基本面数据收集

**必须收集的数据点（若无法获取，标注"数据不可用"）：**

### 营收与增长
```
- 最近 4 个季度 Revenue（QoQ + YoY 增长率）
- 最近 3 年 Annual Revenue + CAGR
- Revenue Guidance（管理层给出的下季/年度预期）
- 收入结构（产品线/地区分布，集中度风险）
```

### 盈利能力
```
- Gross Margin（毛利率）及趋势（扩张/收缩）
- Operating Margin / EBITDA Margin
- Net Income Margin
- Free Cash Flow（FCF）及 FCF Margin
- EPS（摊薄后）及 YoY 增长
```

### 资产负债
```
- Total Debt / Total Equity → Debt/Equity Ratio
- Net Debt（总债务 - 现金）
- Interest Coverage Ratio（EBIT / 利息费用）→ <3x 为警戒线
- Current Ratio（流动资产/流动负债）→ <1 为流动性风险
```

### 估值指标
```
- P/E（市盈率）：当前 vs 5年历史区间 vs 同行
- Forward P/E（基于下年预期）
- P/S（市销率）：适用于亏损成长股
- EV/EBITDA：适用于周期股和传统行业
- PEG = P/E / EPS增长率 → <1 为相对低估信号
- P/FCF（市现率）
```

### 运营指标（行业特定）
```
SaaS：ARR/NRR增长、Churn Rate、Rule of 40
电商：GMV增长、Take Rate、活跃用户
半导体：出货量、ASP趋势、库存水位
消费：同店销售增长、客流量
银行：NIM、NPL率、资本充足率
```

## Step 3: 技术面分析

**价格趋势：**
```
- 当前价格 vs 52周高/低
- 相对大盘（S&P 500）过去 1/3/12 个月的超额收益/不足
- 价格位于 50日均线上方/下方
- 价格位于 200日均线上方/下方
- 均线排列：50MA vs 200MA（金叉/死叉状态）
```

**动量指标：**
```
- RSI（14日）：>70 超买 / <30 超卖 / 50附近为中性
- 成交量趋势：近期放量上涨（强）/ 缩量下跌（弱/整理）/ 放量下跌（出货警告）
- MACD：判断短期趋势方向
```

**关键价位：**
```
- 近期支撑位（前期低点/均线/整数关口）
- 近期阻力位（前期高点/整数关口）
- 若已破位或突破，说明意义
```

**技术面信号汇总：**
```
趋势：上升 / 下降 / 横盘
动量：强 / 中 / 弱
风险/回报：当前价位进入的风险比
```

## Step 4: 新闻与催化剂扫描

**近期事件（过去 3 个月）：**

财报：
```
- 最新季报 EPS vs 预期（Beat/Miss/In-line）
- 营收 vs 预期
- 管理层 Guidance 是否调高/调低
- 财报后股价反应（+X% / -X%）
```

分析师动态：
```
- 评级变化（升级/降级/首次覆盖）
- 目标价变化及理由
- 一致预期变化方向
```

内幕交易：
```
- CEO/CFO/董事买卖（过去 6 个月）
- 大规模抛售 vs 增持的信号意义
```

重大事件：
```
- 并购（买方/卖方）
- 回购计划
- 分红变化
- 监管/法律风险
- 产品发布/重大合同
```

## Step 5: 竞争格局与护城河评估

**市场地位：**
```
- 市场份额（绝对值 + 趋势：扩大/稳定/收缩）
- 主要竞争对手（前3名）
- vs 竞争对手的关键财务指标对比
```

**护城河评估（5 类，每类打分 0-3）：**
```
网络效应（用户越多价值越大）：0/1/2/3
转换成本（用户切换代价高）：0/1/2/3
成本优势（规模/技术/资源）：0/1/2/3
品牌定价权（溢价能力）：0/1/2/3
监管许可/专利壁垒：0/1/2/3

总分 ≥10：宽护城河 | 5-9：窄护城河 | <5：无显著护城河
```

## Step 6: 估值分析

**根据公司类型选择估值方法：**

### 成长股（Revenue增长 >20%）：用 DCF + P/S 对比
```
DCF 简化版（5年）：
  - 假设收入 CAGR：X%（基于过去+guidance）
  - 假设终端 FCF Margin：X%（行业成熟后水平）
  - 折现率（WACC）：通常 8-12%
  - 终端增长率：3%
  - 输出：内在价值 vs 当前价格的溢/折价 %

P/S 对比：
  - 当前 P/S vs 历史区间
  - 当前 P/S vs 同类成长股
  - 规则：P/S 除以 Revenue增长率 → <0.5 为相对便宜
```

### 价值股（成熟行业、稳定盈利）：用 P/E + EV/EBITDA
```
- 当前 P/E vs 历史平均 P/E（5-10年）
- 当前 P/E vs 行业平均
- PEG 比率
- 股息收益率（若有）+ 回购收益率
- 合理价格 = 正常化 EPS × 历史平均 P/E
```

### 周期股：用 EV/EBITDA + 周期位置调整
```
- 当前处于周期的哪个阶段
- 用周期低谷 EPS 而非当前 EPS 来估值，避免高点买入陷阱
```

**估值结论：**
```
当前价格：$XXX
公允价值估计：$XXX - $XXX（区间）
上行空间：+X%（乐观假设）/ +X%（基准假设）
下行风险：-X%（悲观假设）
```

## Step 7: 风险因素

**红旗清单（出现 3 个以上需在报告中重点标注）：**

财务红旗：
```
□ 应收账款增速远超收入增速（收入质量差）
□ 库存积压（连续 2 季增加）
□ 经营现金流与净利润严重背离
□ 频繁股权稀释（每年增股 >5%）
□ Debt/EBITDA > 5x
□ 利息覆盖率 < 2x
```

经营红旗：
```
□ 客户集中度 >30%（前3客户）
□ 核心高管离职（CEO/CFO/CTO）
□ 指引连续下调
□ 毛利率持续收缩
```

市场红旗：
```
□ 做空机构报告
□ SEC 调查 / 会计问题
□ 大股东/内部人连续抛售
□ 债券市场信号（CDS价差扩大）
```

# Gotchas

**数据获取**
- 优先使用 SEC EDGAR（官方财报）、公司 IR 页面，不用未经验证的社交媒体数据
- 财报数据注意 GAAP vs Non-GAAP 的区别（科技公司 Non-GAAP 常排除股权激励，需还原对比）
- TTM（过去12个月）数据不等于最新年报数据，要注意对齐口径

**估值陷阱**
- 周期股在利润高峰期 P/E 看起来很低（"价值陷阱"）——要用标准化利润
- 亏损成长股用 P/E 无意义，用 P/S 但要结合增速
- DCF 对折现率极其敏感——变动1%可导致估值相差30%以上，必须做敏感性分析

**技术面**
- 技术面只反映市场情绪和资金面，不能单独决策
- 中国概念股（中概股）的技术指标受 ADR 机制影响，需单独处理

**催化剂**
- 财报 Beat 不等于涨：若已被预期 price in，Beat 后可能反而下跌（"Buy the rumor, sell the news"）
- 内部人抛售需区分：计划性 10b5-1 卖出 vs 非计划性大规模卖出，前者通常无信号意义

**报告局限性**
- 所有分析基于公开信息，报告末尾必须注明"不构成投资建议"
- 护城河评估是定性判断，需标注依据，避免主观臆断
- 若关键数据缺失（如私有竞争对手数据），明确标注假设来源
