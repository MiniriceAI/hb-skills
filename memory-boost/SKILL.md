# 记忆增强

## 核心定位

解决 Claude Code 每次会话都"失忆"的问题。通过 claude-mem MCP 工具管理三类操作：SEARCH（查找历史决策和上下文）、REPORT（生成时间段内的活动报告）、WRITE（将当前会话的关键信息持久化）。这三个模式是独立的，按需触发。

---

## 触发场景

**SEARCH 模式触发**
- 用户说"我之前是怎么决定X的"
- 用户说"上次我们讨论过这个问题，帮我回顾一下"
- 新会话开始时，用户说"先了解一下上下文"
- 需要查找特定项目的历史决策

**REPORT 模式触发**
- 用户说"生成本周/本月的工作报告"
- 用户说"我最近做了什么，帮我总结"
- 用户说"回顾一下过去 [N] 天"
- 周期性总结任务

**WRITE 模式触发**
- 会话结束时用户说"帮我把这次讨论的要点存起来"
- 重要决策刚刚做出，用户说"记录这个决定"
- 用户说"把这个上下文保存，下次会话用"
- 完成重要任务后的自动归档

---

## 工作流

### SEARCH 模式

1. **理解查询意图**
   - 识别用户想找的是：决策记录 / 任务结果 / 讨论上下文 / 代码实现 / 问题解决方案

2. **构建搜索策略**

   生成 2-3 个搜索查询，覆盖不同角度：
   - 主查询：用户原始问题的核心词
   - 近义查询：同义词或相关概念
   - 上下文查询：项目名/时间/相关人物

3. **执行搜索**

   使用 mcp__plugin_claude-mem_mcp-search__search 工具：
   - query: [搜索词]
   - limit: 10-20（初次搜索）
   - orderBy: relevance（相关性优先）

4. **按相关性排序，获取详情**

   - 从搜索结果中选 Top 3-5 条（按相关性分数）
   - 使用 get_observations 获取完整内容
   - 若结果不足，扩展搜索词重试一次

5. **输出搜索结果**（见输出格式）

---

### REPORT 模式

1. **确认报告参数**
   - 时间范围（默认：最近 7 天）
   - 报告类型（周报 / 月报 / 项目报告）
   - 关注维度（全部 / 仅决策 / 仅任务 / 仅问题）

2. **检索时间范围内的所有记录**

   使用 mcp__plugin_claude-mem_mcp-search__search：
   - dateStart: [N天前的日期，格式 YYYY-MM-DD]
   - dateEnd: [今天]
   - limit: 100
   - orderBy: date

3. **按类别分类所有 observations**

   分类标准：
   - 决策类：包含"决定"、"选择"、"方案"、"确定使用"等关键词
   - 任务完成类：包含"完成"、"实现"、"修复"、"发布"等关键词
   - 洞察类：包含"发现"、"意识到"、"总结"、"规律"等关键词
   - 问题解决类：包含"解决"、"修复"、"原因是"、"方法是"等关键词
   - 其他：不属于以上类别

4. **生成报告**（见输出格式）

---

### WRITE 模式

1. **收集待存储内容**

   若用户未明确提供，从当前会话历史中提取以下内容：
   - 重要决策（选择了什么方案及原因）
   - 完成的任务（做了什么，结果如何）
   - 发现的规律或洞察
   - 遇到并解决的问题（问题描述 + 解决方案）
   - 下次需要知道的上下文

2. **格式化为 observation**

   每条记忆条目格式：
   - 类型：[决策/任务/洞察/问题解决/上下文]
   - 项目：[项目名，若适用]
   - 内容：[一段简洁的陈述，包含足够上下文让未来的自己理解]
   - 关键细节：[时间、版本号、文件路径等具体信息]

3. **写入 claude-mem**

   使用对应写入工具。

   写入优先级：
   - 必写：重要决策（尤其是选择了A而不是B的原因）
   - 必写：跨会话需要的上下文
   - 选写：任务完成记录
   - 不写：临时信息、过程中的试错步骤

4. **确认写入成功**，输出已写入条目列表

## Memory Search 语法参考

基础关键词搜索：
  query: "React 状态管理"

按项目过滤：
  project: "my-project"
  query: "部署配置"

按时间范围：
  dateStart: "2024-01-01"
  dateEnd: "2024-03-31"

按类型过滤：
  obs_type: "decision"  （或 task / insight / issue）

组合查询：
  query: "数据库"
  project: "backend-api"
  dateStart: "2024-10-01"

按相关性排序（默认）：
  orderBy: "relevance"

按时间排序（报告时用）：
  orderBy: "date"

## Gotchas

### claude-mem 工具未连接
- 症状：调用 mcp__plugin_claude-mem_mcp-search__search 返回连接错误
- 处理：告知用户"claude-mem MCP 工具当前未连接，无法执行记忆操作。请检查 MCP 服务是否运行"
- 不要尝试用对话历史模拟持久记忆

### 搜索结果不相关
- 症状：搜索词命中的 observation 与用户问题无关
- 处理：换用更具体的词 → 尝试时间范围过滤 → 若仍无结果，如实告知

### WRITE 时内容过多
- 单次写入控制在 5 条以内
- 若当前会话内容丰富，按重要性排序，只写 Top 5
- 告知用户"本次会话内容较多，已优先存储以下重要决策和上下文，其余为过程信息未存储"

### REPORT 时时间范围内记录为空
- 先扩大时间范围（翻倍）
- 若仍为空，告知用户"在[日期范围]内没有找到记录，可能是 claude-mem 在该时段没有活跃写入"
- 建议用户查看是否有更早的记录

### observation 内容不够具体
- 写入时避免模糊表述
- 差："讨论了认证方案"
- 好："决定使用 JWT + Redis 存储 session，原因：无状态 API + 需要支持多端登录，文件位置：src/auth/middleware.ts"
- 具体信息（版本号、文件路径、命令、URL）应尽量保留在 observation 中
