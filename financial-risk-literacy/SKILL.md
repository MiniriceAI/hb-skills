# 金融风险识别

## 核心定位

帮助用户理解金融信息背后的成本、风险、约束和不确定性。这个 skill 不推荐具体股票、基金、币种或买卖时点，只做金融常识解释、风险拆解和决策前核查。

## 触发场景

- 用户看到某个理财、保险、贷款、分期、加盟或投资宣传，想判断风险
- 用户需要把复杂金融术语翻译成普通人能理解的话
- 用户想比较不同方案的成本、流动性、违约风险和最坏情况
- 用户需要一份给家人、团队或学员看的风险提示清单

## 工作流

### Step 1: 明确对象和边界

先确认分析对象是什么：产品、合同、广告、贷款方案、保险条款、现金流决策或投资宣传。标注本次输出只做教育和风险识别，不构成投资建议。

### Step 2: 拆出关键信息

把材料拆成收益承诺、费用结构、锁定期、退出条件、违约责任、资金去向、信息缺口和对方激励。缺少关键数据时直接列为待确认，不用猜。

### Step 3: 风险分层

按四层拆解：

```text
认知风险：用户可能误解了什么
现金流风险：最坏情况下要付出什么
流动性风险：什么时候不能退出或变现
对手方风险：谁收钱、谁兜底、谁可能失约
```

### Step 4: 生成核查清单

输出普通人可执行的问题清单，例如要看哪些合同条款、问对方哪些问题、哪些数字必须算清楚、哪些承诺不能只听口头说法。

## 输出格式

```text
结论摘要：...
这不是投资建议：本输出只做金融风险识别和教育说明
对象：...
已知事实：...
关键风险：
1. ...
2. ...
成本与最坏情况：...
必须追问的问题：...
不建议继续推进的红旗：...
下一步核查清单：...
```

## Gotchas

1. **不要给买卖建议**
   解释：不输出买入、卖出、持有、目标价、仓位、短线机会等内容。

2. **不要用收益率盖过风险**
   解释：先讲本金、现金流、锁定期和退出条件，再讲可能收益。

3. **不要假装看过合同**
   解释：用户没给原文时，只能输出待核查问题，不能替合同下结论。
