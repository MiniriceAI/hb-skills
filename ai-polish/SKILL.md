---

## 核心定位

AI写出的中文内容有极度固定的语言模式和节奏，训练有素的读者一眼就能识别。这个技能的目标：**诊断AI写作的具体病症，然后用真人语言替换它**。不是微调，是手术。

---

## 触发场景

- "帮我润色一下，去掉AI味"
- "这篇文章读起来太像AI了，改一下"
- "让这段话听起来更像我说的"
- "检查一下有没有AI写作的痕迹"

---

## 工作流

### 步骤1：全文扫描（诊断阶段，不修改，只标注）

在修改任何内容之前，先完成全部诊断，按以下清单逐项检查并标注问题类型：

---

**A类：万能开头结尾词（出现即删）**

以下词语出现即删除或替换，不需要分析：
- `总的来说` / `综上所述` / `总而言之`
- `值得注意的是` / `不难发现` / `显而易见`
- `随着X时代的到来` / `在当今社会` / `当下这个时代`
- `越来越多的人开始` / `不少人都有这样的体验`
- `希望本文能对你有所帮助` / `欢迎关注点赞收藏`
- `这是一个值得深思的问题` / `我们需要认真思考`
- `在这个快节奏的时代` / `生活在这个时代的我们`

---

**B类：过度并列结构（最高频AI特征）**

特征：连续3个以上格式完全相同的句子或段落。

诊断标准：选任意5句连续内容，若每句起始词都是"首先/其次/再次/最后"或"第一/第二/第三"，判定为B类问题。

```
错误示范：
首先，我们要做好准备工作。
其次，我们需要制定详细的计划。
再次，我们应当严格执行方案。
最后，我们总结复盘积累经验。

真人写法：
准备工作很多人跳过了，这是后面反复返工的根源。
计划不需要完美，能执行的计划比精致但搁置的计划有价值得多。
执行中发现的问题，往往比计划阶段预想的更奇怪——这时候怎么应对比计划本身更重要。
```

---

**C类：机械过渡词（替换为自然连接）**

过度使用以下词时需要重写段落间的连接：
- `不仅如此` / `与此同时` / `除此之外` / `另外`
- `更重要的是` / `值得一提的是` / `需要指出的是`
- `总体而言` / `从整体上看` / `综合来看`
- `对于这个问题` / `针对这一现象` / `就这一点而言`

替换方向：要么直接连接（删掉过渡词，两段直接相邻），要么用具体的因果关系或场景替代过渡词。

---

**D类：空洞形容词（用具体细节替换）**

诊断：以下词出现密度高时判定为D类问题：

过于宏大：`深刻` / `显著` / `卓越` / `非凡` / `重要` / `关键` / `本质` / `内在逻辑`

过度保险：`一定程度上` / `某种意义上` / `相对来说` / `从某种角度看` / `不同程度地`

```
错误示范："这个方法非常有效，对我们产生了深远的影响。"
真人写法："用这个方法，我三天内解决了卡了两个月的问题。"
```

规则：删掉形容词，用具体数字、时间或结果替代。

---

**E类：句子节奏机械（最难改但最影响感受）**

AI生成文章的句子长度高度均匀（每句约15-25字），节奏像机器人朗读。真人写作的特征：长短交替，偶尔一句话就是一句话。

诊断标准：选任意连续5句，若字数差异都在5字以内，判定节奏机械。

```
错误示范（每句均匀约20字）：
这个问题需要我们认真对待，因为它影响着我们的生活。
很多人在面对这种情况时都会感到困惑，不知该如何处理。
我们可以采取一些有效的措施来改善这种情况，提升质量。

真人写法（长短不一）：
这个问题很麻烦。
不是说不能解决，而是很多人根本不知道从哪下手——确认问题在哪才是第一步。
我后来试了这个方法，解决了。
```

---

**F类：空洞升华结尾（直接删掉或换实质内容）**

```
典型AI结尾（删掉）：
"生活就是如此，需要我们不断探索和前行。"
"愿我们都能在这条路上找到属于自己的答案。"
"这不仅仅是一个选择，更是一种态度。"
"让我们共同努力，迈向更美好的明天。"

真人结尾：直接说要做什么，或留一个真实问题，或在自然的地方停下。
```

---

**G类：第一人称缺失（添加真实视角）**

AI倾向于用"我们"泛指，或完全回避第一人称。真人写作有具体的个人视角。

```
错误示范："人们普遍认为，在面对挑战时需要保持积极心态。"
真人写法："我以前也这么想，直到第三次失败后才发现这句话的问题在哪。"

错误示范："我们需要改变这种观念，重新审视自己的选择。"
真人写法："我是怎么改变的：[具体一件事]让我意识到[具体洞察]。"
```

---

### 步骤2：打标汇总

完成扫描后，汇总问题清单：
- A类：X处
- B类：X处（举1个典型例子）
- C类：X处
- D类：X处
- E类：整体节奏评级（轻微/中度/严重）
- F类：有/无
- G类：有/无

给出总体AI味程度评级：轻微/中度/明显/严重

### 步骤3：分段重写

按批注逐段修改：
- A类：直接删除，不需要替换文字
- B类：打破并列，改为因果关系/转折逻辑/叙事推进
- C类：删掉过渡词，重写段落连接逻辑
- D类：删掉形容词，找具体细节（数字/时间/结果）替代
- E类：重新切割句子，主动制造长短对比
- F类：删掉最后1-2段空洞升华，换一句具体的话
- G类：加入第一人称具体视角和经历

### 步骤4：朗读节奏检查

修改完成后，模拟朗读全文：
- 在哪里会自然停顿？那里应该是段落结束或句号
- 哪句话读起来像在背稿而非说话？重写那句
- 整体是否有快慢起伏？太均匀则节奏仍有问题

## 输出格式

### 短文（1000字以内）

直接输出修改稿，末尾附：
```
【修改说明】
发现并处理的AI特征：
- [类型] × [X处]：[举一个典型改法示例]
- [类型] × [X处]：[举一个典型改法示例]
主要改动方向：[一句话总结]
```

### 长文（1000字以上）

先输出诊断报告，再输出修改稿：
```
【AI特征诊断报告】
A类（套话）：[X处，举例]
B类（并列）：[X处，举例]
C类（过渡）：[X处，举例]
D类（空洞形容）：[X处，举例]
E类（节奏）：[轻微/中度/严重]
F类（结尾升华）：[有/无]
G类（视角缺失）：[有/无]

总体AI味程度：[轻微/中度/明显/严重]
主要问题：[最需要解决的1-2类问题]
---
[修改稿]
```

## Gotchas

**最常被忽略的AI特征**
节奏问题（E类）是最难被读者描述但最影响感受的问题。读者说"这篇文章读起来不舒服"，80%的原因是节奏机械，但他们描述不出来。修节奏比改用词更重要，优先级最高。

**不能过度修改的情况**
- 专业信息需要准确传达时，不能为了"自然"而改变信息准确性
- 正式报告/商业文案类内容，适当保留规范表达，不必完全口语化
- 若原文本身质量高只有少量AI特征，不要大改，只修问题点

**常见误操作**
- 把所有"首先其次"都改掉：有时列举逻辑确实需要顺序词，保留逻辑必要的，删除装饰性的
- 把所有长句切短：真人写作也有长句，关键是要有变化，不是都要短
- 加入太多"我"的主观感受：科普/分析类内容不需要太多第一人称，根据内容类型判断

**判断改得够不够的标准**
改完后问：如果这段话是一个真实的人当面对你说的，你会觉得奇怪吗？
会觉得奇怪 = 还有AI味，继续改。
完全自然 = 完成。
