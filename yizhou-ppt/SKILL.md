# 核心定位

将原始想法或素材转化为结构完整、叙事清晰的演示文稿。不是简单地把内容分页，而是为每个 PPT 设计一条说服或传达信息的叙事弧线，使听众能跟随逻辑走向行动或决策。

---

# 触发场景

- 用户说"帮我做一个 PPT"或"帮我整理一个汇报"
- 提供了某主题文字素材，需要转为演示格式
- 需要投资路演、培训课件、工作汇报、年终总结、产品发布等场景
- 现有 PPT 结构混乱，需要重构逻辑

# 工作流

## Step 1: Intake（信息收集）

在动笔前，明确以下 5 个变量，缺一不可：

| 变量 | 说明 | 若用户未提供 |
|------|------|------------|
| 目的 | persuade / inform / train / inspire | 追问："这个 PPT 最终要让听众做什么？" |
| 受众 | 决策者 / 技术团队 / 外部客户 / 大众 | 追问："谁来看这个 PPT？他们的背景是？" |
| 时间限制 | 演讲时长（分钟） | 默认按 20 分钟设计 |
| 页数预算 | 目标页数上限 | 按每页 1.5 分钟推算（20 分钟 ≈ 13 页） |
| 核心信息 | 听众离场时必须记住的 1 句话 | 追问或根据素材提炼 |

## Step 2: 选择叙事模型

根据目的选择对应的叙事结构：

### 模型 A：融资/商业 Pitch Deck（说服决策者）
```
1. 钩子（1页）：用数字或问题制造紧迫感
2. 问题（1-2页）：痛点有多真实、多大
3. 解决方案（1-2页）：你怎么解决
4. 产品/服务演示（2-3页）：怎么运作
5. 市场规模（1页）：TAM/SAM/SOM
6. 商业模式（1页）：怎么赚钱
7. 牵引力/证据（1-2页）：已有成绩
8. 团队（1页）：为什么是你们
9. 需求/CTA（1页）：要什么，给什么
```

### 模型 B：培训课件（传授技能）
```
1. 学习目标（1页）：学完能做什么
2. 为什么重要（1页）：不学的代价
3. 核心概念（3-5页）：每页一个概念
4. 案例/演示（2-3页）：看真实例子
5. 练习框架（1-2页）：怎么上手
6. 常见误区（1页）：避免踩坑
7. 总结与行动（1页）：明天就能做的事
```

### 模型 C：工作汇报/项目报告（向上沟通）
```
1. 结论先行（1页）：BLUF（Bottom Line Up Front）
2. 背景（1页）：为什么做这件事
3. 做了什么（2-3页）：关键行动
4. 数据结果（2-3页）：成果如何量化
5. 问题与风险（1页）：诚实面对挑战
6. 下一步（1页）：接下来的计划
7. 需要支持（1页）：请求决策或资源
```

### 模型 D：产品发布/演讲（激励感召）
```
1. 开场故事或冲突（1-2页）：让人产生共情
2. 世界在改变（1页）：趋势/转折点
3. 介绍产品（2-3页）：它做什么、为谁做
4. 现实案例（2页）：真实用户/场景
5. 愿景（1页）：更大的意义
6. 行动号召（1页）：怎么参与
```

## Step 3: 分页规划

为每一页生成规划卡，格式如下：

```
页码: 3
标题（≤8字）: 市场规模足够大
核心信息（1句话）: 国内知识付费市场规模达 2023 年 677 亿，年增长 18%
最多 3 个要点:
  · 677 亿市场，年增 18%
  · 用户付费意愿提升，客单价从 200→450 元
  · 头部玩家仅占 15%，碎片化市场待整合
视觉建议: 柱状图展示市场规模 3 年增长趋势，或大数字突出显示
```

分页规划完成后，**必须向用户确认整体结构**，再进入内容撰写。

## Step 4: 逐页内容撰写

每页按以下模板输出：

```
[第 X 页：标题]

页面核心信息：
[这页要传达的唯一主旨，1-2句]

正文内容：
· 要点 1：[具体、可量化、避免空话]
· 要点 2：[...]
· 要点 3：[可选]

视觉指引：
[具体描述图表类型、数据范围、或图片情绪基调]

备注/数据来源：
[需要核实的数据，或建议插入的案例]
```

**密度规则（严格执行）：**
- 每页只有一个核心信息
- 正文要点 ≤ 3 条，每条 ≤ 15 字
- 标题必须是结论或问题，不是话题（"销售额增长 40%"而非"销售数据"）
- 禁止使用超过 4 行的段落文字

## Step 5: 演讲备注

每页补充演讲备注，格式：

```
[备注 第X页]
开场语：[如何过渡到这页]
关键讲解点：[页面文字背后要口头补充的信息]
互动时机：[是否在此停顿、提问或展示 demo]
时间分配：约 X 分钟
```

## Step 6: 最终审查清单

```
□ 第 1 页的钩子是否在 10 秒内抓住注意力？
□ 每页核心信息是否能在 5 秒内被读懂？
□ 是否存在超过 3 个要点的页面？（如有，拆分）
□ 数据是否来源可靠，是否注明？
□ 视觉建议是否与内容一致？
□ 结尾是否有明确的 CTA（行动号召）？
□ 总页数是否在时间预算内？
□ 受众是否在整个结构中被考虑？
```

# 输出格式

**交付物结构：**

```
[PPT 概要卡]
- 标题：
- 目的：
- 受众：
- 时长/页数：
- 核心信息（一句话）：
- 叙事模型：

[分页规划表] — 先输出，等用户确认

[逐页内容]
第 1 页 ...
第 2 页 ...
...

[演讲备注]

[审查清单]
```

若用户只需要快速大纲，输出简化版：每页一行，格式为 `页码 | 标题 | 核心信息`。

# Gotchas

**叙事陷阱**
- 开场页不要写"目录"——改用问题或冲突，先抓住听众注意力
- "关于我们"公司背景页永远不要放在第 2 页，听众还未关心你是谁
- 结尾不能只是"谢谢"——必须有一个清晰的下一步或 CTA

**内容密度陷阱**
- 用户常要求"把所有信息都放进去"——坚持密度规则，并解释原因：PPT 是演讲辅助，不是文档
- 一页超过 3 个要点，听众记不住任何一个
- 长段文字意味着听众在阅读，不在听你讲

**数据呈现陷阱**
- 百分比变化要同时给出基数（"增长 200%"在 5 单到 15 单和 500 到 1500 是不同概念）
- 图表的标题应该是结论，不是描述（"2024 年用户增长 3 倍"，不是"用户增长趋势图"）

**格式陷阱**
- 视觉建议只是建议，不要生成 PowerPoint 文件——描述清楚让用户自行制作或交给设计
- 若用户想要中文内容，所有要点保持中文；若英文 PPT，坚持英文，不要混排

**常见误区**
- 培训类 PPT 不要只讲理论——每个概念后必须紧跟一个可操作的例子
- 汇报类 PPT 不要把结论放在最后——上级时间有限，结论要在第 1 页
- Pitch Deck 的团队页要讲"为什么你们有资格解决这个问题"，不是简历罗列
