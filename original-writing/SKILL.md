## 核心定位

不是"写一篇关于X的文章"，而是找到**只有这个角度才能这样写的理由**，然后用专业结构把它呈现。核心：差异化角度 + 真实洞察 + 让人读完的内部张力。

---

## 触发场景

- "帮我写一篇关于[话题]的长文"
- "我需要一篇3000字以上的原创文章"
- "帮我写一篇深度内容发公众号/知乎"
- 明确需要完整长文初稿的内容创作需求

---

## 工作流

### 步骤1：受众精准画像（不可跳过）

在动笔前，必须明确：
- **核心读者是谁**：不是"25-35岁职场人"，而是"在大厂做了3年产品经理、最近开始焦虑天花板的人"
- **他们现在最大的困惑或痛点是什么**
- **他们对这个话题已经知道什么**（避免解释他们已知的内容）
- **他们看完这篇文章希望得到什么**：认知改变/情绪共鸣/方法论/信心

输出一句话受众定义：`[具体人群]，正在经历[具体处境]，希望得到[具体收获]`

### 步骤2：角度差异化（最关键步骤）

列出关于这个话题**最常见的3-5种写法**，然后问：
- 这些文章的共同盲点是什么？
- 有哪个视角是"显而易见但没人这样写过"的？
- 有哪个真实案例可以颠覆主流观点？

**4种差异化角度来源**：
1. 反直觉：大家都说X是好事，但从Y角度看其实是问题
2. 具体化：大家都在说抽象原则，用一个最能说明问题的具体故事替代
3. 时间位移：这个话题5年前/10年后看是什么样的？
4. 跨领域嫁接：把A领域的成熟框架用来解释B领域的现象

**记录角度决策**：写下"我选这个角度，因为[具体原因]，这篇和其他同类文章的本质区别是[X]"

### 步骤3：结构设计（张力弧线）

一篇能读完的长文需要内部张力，而不是平铺直叙：

```
开篇（建立悬念/冲突）
    ↓
第一转折（打破读者预期）
    ↓
核心论证展开（证据+案例）
    ↓
第二转折（更深洞察，或反驳潜在质疑）
    ↓
结局（解决冲突，给出清晰结论）
    ↓
行动召唤
```

**字数分配**（以5000字为例）：
- 开篇钩子：150-300字（必须在100字内建立张力）
- 背景铺垫/问题界定：400-600字
- 核心论证段落1：800-1000字（最重分量）
- 核心论证段落2：800-1000字
- 转折/深化：600-800字
- 案例/故事支撑：600-800字
- 结论+行动：300-500字

### 步骤4：开头钩子写作（前100字生死线）

**5种钩子模式（选一种）**：

1. **悖论开场**："[广泛认同的观点]。但[具体数据/案例]告诉我们，事实恰好相反。"
2. **场景代入**：直接把读者放入一个他认识的具体场景，不解释，直接开始
3. **数字冲击**："[让人意外的具体数字]——这是[令人不安的现象]。"
4. **问题挑衅**：一个让目标读者无法不点头的痛点问题
5. **故事开场**：从一个具体人物的具体时刻开始——不是"据说有个人"，是"2023年秋天，[具体情节]"

**钩子自检标准**：
- 读完前100字，读者是否会想"然后呢？"
- 钩子是否指向了文章核心矛盾而非旁枝末节？
- 是否避免了烂俗开场："随着X时代的到来...""在当今社会...""越来越多的人..."

### 步骤5：逐段撰写规则

**每个段落的内部结构**：
- 论点句（明确这段说什么）
- 证据/案例（支撑论点，越具体越好）
- 分析（为什么这个证据支持论点）
- 过渡（连接到下一段，要有逻辑而非仅靠转折词）

**具体性原则（强制执行）**：
- 不写"很多人都..."，写"2024年一项针对1200名职场人的调查显示..."
- 不写"有一个著名案例"，写出具体的公司/人名/时间/数字
- 不写"这说明了X的重要性"，写"这意味着，如果你[具体情况]，你需要[具体行动]"

### 步骤6：结尾与行动召唤

结尾不是"总结全文"，而是：
- 回应开篇建立的悬念（呼应，但不是重复）
- 给读者一个**清晰的下一步行动**（越具体越好）
- 留下一个让人思考的问题或洞察（让文章在读完后还有余韵）

## 输出格式

### 写作前确认（先输出，用户确认后再写正文）

```
【文章策划】
目标读者：[一句话精准描述]
核心角度：[这篇文章和其他同类文章的本质区别]
张力来源：[读者带着什么问题进来，离开时得到什么答案]
预计字数：[X]字
钩子类型：[选用哪种开头模式]

结构草案：
  开篇：[核心矛盾/场景]（约[X]字）
  第一部分：[主题]（约[X]字）
  第二部分：[主题]（约[X]字）
  转折：[深化方向]（约[X]字）
  结尾：[行动召唤方向]（约[X]字）

确认后开始撰写。
```

### 正文输出

完整文章，包含：
- 标题（主标题 + 可选副标题）
- 正文（按结构分段，适当使用小标题）
- 末尾注明实际字数

## Gotchas

**必须主动避开的AI写作病症**

1. 正反合结构陷阱：文章前半段说A的好，后半段说A的问题，结尾说要平衡——这是最无聊的结构，不产生任何洞察
2. 抽象原则堆砌：全文都是"要注重X、培养Y、建立Z"，没有一个具体案例
3. 假装深度：用复杂词汇包装浅显观点，读者读完什么都没学到
4. 过度列表化：把所有内容写成第一点/第二点/第三点，丧失叙事流动性
5. 万能结尾："希望这篇文章对你有所帮助，欢迎分享给身边的朋友" → 直接删掉

**角度选择的死亡信号**
- 角度是"[话题]很重要" → 废弃，找更具体的切入点
- 角度是"[话题]有优缺点" → 废弃，选一个立场
- 角度是"X和Y都要兼顾" → 废弃，这不是观点，是回避

**字数的正确处理方式**
- 不要为凑字数加废话，宁可3000字精华也不要5000字稀释
- 若用户指定字数，通过增加案例深度、数据支撑来扩展，而非重复论点

**真实性原则**
- 案例数据若无法确认，加"据报道"或给出大概范围，不要伪造精确数字
- 专家引用必须是真实存在的观点，不要虚构引用来增加权威感
