---

## 核心定位

洗稿不是同义词替换，而是**理解参考内容为什么有效，然后用自己的语言和素材重新表达同样的有效性**。合格标准：结构可学，逻辑可用，但表达、案例、比喻全部是新的，任意15字连续片段与原文不重合。

---

## 触发场景

- "帮我把这篇文章改成我的风格"
- "参考这篇爆款，帮我写一篇类似的"
- "基于这个内容框架，用不同的例子重写"
- "这篇文章很好，我需要一个不雷同的版本"

---

## 工作流

### 步骤1：解构参考内容（阅读分析阶段）

不要直接开始改写，先做完整解构：

**1.1 提取核心主张**
- 这篇内容的核心论点是什么？（一句话）
- 作者的核心立场是什么？（支持/反对/揭示了什么）

**1.2 绘制逻辑骨架**
```
论点1 → 支撑证据（类型：数据/故事/对比/权威） → 小结论
论点2 → 支撑证据 → 小结论
...
总结论
```

**1.3 爆款因素诊断**
识别这篇内容为什么有效（找出1-2个主要因素）：

| 因素 | 是否是本文主要因素 |
|-----|----------------|
| 情绪钩子（触发愤怒/感动/共鸣/恐惧） | |
| 数据冲击（意外的数字或研究结论） | |
| 反常识（颠覆主流认知） | |
| 故事代入（主角情境与读者高度重叠） | |
| 实用价值（直接可操作的方法） | |
| 身份认同（说出某类人想说没说的话） | |

**1.4 语言风格标注**
- 句子长度：短促/长句/混合
- 人称：第一人称亲历/第三人称客观/对话体
- 语气：严肃/亲切/犀利/幽默
- 标志性表达：找出3-5个最有特色的句式

### 步骤2：明确改写边界

**必须保留（结构层）**：
- 文章的整体逻辑结构（起承转合的节奏）
- 论点与论点之间的推进关系
- 使内容有效的核心机制（若参考用"问题-方案"有效，保留这个框架）
- 段落的功能分工（哪段建立共鸣，哪段给解决方案）

**必须改变（表达层，零容忍）**：
- 所有句子的具体表达
- 所有案例和例子（必须替换为不同的案例）
- 所有比喻和类比
- 标题和所有小标题
- 开头和结尾的具体内容
- 若引用数据，必须来自不同来源

**查重硬标准**：任意15个连续汉字不得与原文重合

### 步骤3：目标风格设定

改写前必须明确目标声音（若用户未指定，追问）：
```
目标声音：亲历者 / 观察者 / 教练 / 朋友 / 专家
语气温度：冷静客观 / 温暖亲近 / 犀利直接 / 幽默轻松
句子风格：短句冲击 / 长短交替 / 散文流动
人称选择：我 / 你 / 我们 / 第三人称
```

若用户提供风格范本（某人的文章/某平台的文章），提取其标志性句式作为参照。

### 步骤4：分块改写执行

按段落/部分逐块改写，每块完成后检查：
1. 核心论点是否保留？
2. 表达是否100%是新的？
3. 风格是否符合目标声音？
4. 案例是否已替换？

**改写的4种核心技术**：

**更换例子**：参考内容用A公司举例 → 找一个能说明同样道理的B公司，或换成个人生活场景。替换后效果应不弱于原例子。

**更换比喻**：参考内容把X比作"马拉松" → 用另一个同样有效的比喻，或干脆改为直接描述而不用比喻。不要强行找替换，如果找不到更好的比喻，换一种表达方式。

**重构数据支撑**：参考内容引用了研究数据 → 寻找说明同样现象的不同数据，或改用逻辑推理代替数据。

**重构开头**：不论原文用什么开场类型，用完全不同的类型替代（原文用故事 → 用数据；原文用问题 → 用反常识陈述）。

### 步骤5：全文质检

**查重检验**：
将改写稿和原文各取5个段落并排，逐句检查是否有句式雷同。
重点检查：过渡句、列举句、总结句（这些最容易不知不觉保留原文结构）

**有效性验证**：
- 改写稿是否保留了原文让人想读下去的核心张力？
- 原文的情感共鸣点是否在改写稿中同样存在（可以用不同方式实现）？

## 输出格式

### 改写前分析报告（先输出，确认后再开始改写）

```
【参考内容解构】

核心主张：[一句话]
逻辑骨架：
  1. [论点1] → [证据类型] → [小结论]
  2. [论点2] → [证据类型] → [小结论]

爆款核心因素：[1-2个最主要的]
原文语言风格：[简短描述]

【改写方案】
目标声音：[设定]
结构调整：[若保留，注明"保留原结构"；若调整，说明如何调整]
案例替换计划：
  - 原文案例[描述] → 替换为[新案例方向]
  - 原文案例[描述] → 替换为[新案例方向]
开头策略：[用什么类型的钩子，与原文不同]

确认方案后开始改写。
```

### 改写稿输出

完整改写稿，末尾附：
```
【改写说明】
结构保留：[说明保留了什么逻辑结构]
主要改变：[表达、案例、比喻的改变方向]
案例对照：原文[X] → 改写[Y]
```

## Gotchas

**同义词替换不是洗稿**
"美好的未来"→"光明的前景"不是改写，这是替换。真正的改写是把"通过持续努力可以实现目标"改写成一个具体的反例故事，然后从故事里得出同样的结论。

**最容易暴露的位置**
- 列举项的数量（原文"3个方法"→应改为不同数量）
- 过渡语（"不仅如此""更重要的是"容易照搬）
- 文章的节奏节点（转折点的位置往往会被无意识地复制）
- 结论段（大量改写者到结尾忘了改，直接用原文总结）

**改写质量下降的常见原因**
- 为了不重复而用词生僻 → 应该换表达方式，不是换生僻词汇
- 换了案例但说明力不如原文 → 宁可用相似案例讲清楚，也不用牵强案例敷衍
- 原文的情感共鸣被改掉了 → 改写时主动问"原文让读者有什么感受，我的改写保留了这个感受吗"

**不适合改写的情况**
- 原文包含作者独家亲身经历或独家访谈 → 只能参考结构，不能改写内容
- 原文核心数据是作者原创研究 → 必须替换为其他数据来源，不能沿用数字
